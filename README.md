# Caramel Digital Studio Social Media Landing Page

A branded static landing page created by **Caramel Digital Studio** to centralize links to Facebook, X, Discord, and other social media profiles, contact points, and key online destinations in one clean, mobile‑friendly page.

## Overview

This repository contains a lightweight social media landing page project intended to serve as a central link hub. It is designed for use in social media bios, profile links, campaign pages, creator pages, or branded contact hubs where visitors need one clear place to find all important links.

The project is suitable for personal brands, freelancers, online communities, and small business profiles that want a more polished destination than a plain list of links.

## Purpose

The purpose of this project is to provide a simple branded landing page that can:

- Gather multiple profile and destination links in one place.
- Improve navigation from social media to websites, communities, bookings, contact options, or other platforms.
- Present a stronger and more consistent brand identity.
- Create a cleaner user experience for mobile visitors coming from social apps.

## Site Type

This is a **static landing page website**. It is not a web application or CMS‑driven platform. It is intended to function as:

- A social media link hub.
- A branded “link in bio” page.
- A profile landing page.
- A campaign or community redirect page.
- A lightweight static marketing page.

## Features

Typical features of this type of project include:

- A headline and short brand or profile introduction.
- A logo, profile image, or visual identity.
- Buttons or link cards for major social platforms.
- Links to website, portfolio, contact page, shop, booking form, newsletter, or community invite.

## Tech Stack

This project is intended to use a simple static front‑end setup:

- **HTML** for structure.

## Running Locally

Because this is a static site, no build process is required.

To run locally:

1. Clone the repository.
2. Open the project folder.
3. Open `index.html` directly in a browser, or serve it with a lightweight local server.

Example with Python:

```bash
python -m http.server 8000
```

Then open:

```text
http://localhost:8000
```
